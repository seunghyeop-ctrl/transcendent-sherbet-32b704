# 동기화 가이드

## 지금 상태

이 프로젝트는 이제 **로컬 Git 저장소**입니다.

현재 커밋:
- `cbfce45` `Initialize portable prompt extractor repo`

하지만 아직 **원격 저장소(remote)** 가 없습니다.
그래서 지금은:
- 집 맥북에서 수정
- 회사 맥북이나 윈도우에서 자동 반영

이 **되지 않습니다.**

## 자동처럼 이어서 쓰는 방법

가장 좋은 방법은 GitHub private repo를 하나 만드는 것입니다.

### 1. GitHub에 private repository 생성

예시 이름:
- `grommy`
- `prompt-extractor`

### 2. 현재 프로젝트에 remote 연결

```bash
cd "/Users/seunghyeop/Documents/Claude/Projects/프롬프트 추출 및 분류 작성 자동화 프로그램 개발"
git remote add origin <GITHUB_REPO_URL>
git push -u origin main
```

### 3. 회사 맥북 / 윈도우에서 받기

```bash
git clone <GITHUB_REPO_URL>
cd <repo-folder>
```

### 4. 작업 흐름

집에서:
```bash
git add .
git commit -m "작업 내용"
git push
```

회사에서:
```bash
git pull
```

윈도우에서도 똑같습니다.

## 새 컴퓨터에서 꼭 다시 넣어야 하는 것

Git으로 자동 동기화하면 안 되는 값은 제외돼 있습니다.

새 컴퓨터에서 별도로 준비:
- Gemini API Key
- `credentials.json`
- Google Sheet ID / 워크시트 설정

즉:
- **코드는 Git으로 동기화**
- **비밀값은 각 컴퓨터에서 다시 설정**

## 왜 이렇게 해야 하나

이유:
- API 키와 서비스 계정 파일은 Git에 올리면 안 됨
- 결과 CSV/XLSX도 컴퓨터마다 다를 수 있음
- 캐시/임시 파일은 OS마다 달라짐

그래서 `.gitignore`로 이런 파일들은 제외해둔 상태입니다.

## 윈도우/맥 같이 쓰는 경우

지금 코드는 OS 독립형으로 한 단계 정리되어 있습니다.

설정 폴더 기본값:
- macOS: `~/Library/Application Support/PromptExtractor`
- Windows: `%APPDATA%\\PromptExtractor`
- Linux: `~/.config/PromptExtractor`

즉 소스 코드는 같은 저장소를 써도 되고,
각 기기에서 설정/비밀값만 따로 가지면 됩니다.
